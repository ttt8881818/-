# ハコリノベ 業務自動化プロジェクト 引き継ぎドキュメント

## プロジェクト概要

ハコリノベ横浜山下公園のリノベーション設計業務を自動化するプロジェクト。
主にお客様へのヒアリング〜商品選定〜見積り依頼までの流れを自動化する。

## 最優先で解決すべき問題

### フォーム送信後にNotionに保存されない

**原因**
フォームがVercel（外部サーバー）から配信されるようになったことで、Anthropic APIの認証が通らなくなっている。チャット上ではClaude.aiの認証が使えていたが、外部URLからは同じ方法では動かない。

**解決策**
Vercelの環境変数にAnthropicのAPIキーを設定して、フォームからAPIを叩けるようにする。

**具体的にやること**
1. Vercelの環境変数に `ANTHROPIC_API_KEY` を設定
2. index.htmlのAPIコール部分をVercelのサーバーレス関数経由に変更
3. Notion MCP経由での保存が動くか確認

## フォームの仕様

### 全体の流れ

```
お名前入力
　↓
交換する設備を選択（複数選択可）
　キッチン / ユニットバス / 洗面台 / トイレ
　↓
選択した設備のみ質問が表示される
　↓
送信 → Notionに自動保存
```

### キッチン（Q1〜Q11）

```
Q1  スタイル（複数選択可）：壁付け / 対面 / ペニンシュラ / アイランド / Ⅱ型 / L型
Q2  希望幅：〜1800mm / 2100mm / 2400mm / 2550mm / 2700mm〜 / 自由入力
Q3  コンロ種類：ガス / IH
Q4  コンロ口数：2口 / 3口
Q5  グリル：不要 / 片面焼き / 両面焼き
Q6  食洗機：不要 / 必要（国産） / 必要（海外製）
Q7  ホース引き出し：必要 / 不要
Q8  自動水栓：必要 / 不要
Q9  浄水器：必要 / 不要
Q10 パントリー：必要 / 不要
Q11 画像添付・その他ご要望（自由入力＋画像添付可）
```

### UB（Q1〜Q4）

```
Q1 サイズ：1216 / 1418 / 1620 / 1624 / 現状のまま
　 ※ 後ろ2桁が奥行き。数字が大きいほど足を伸ばして浸かれる
Q2 浴室暖房乾燥機：必要 / 不要
Q3 ランドリーパイプ：0本 / 1本 / 2本
Q4 その他ご要望（テキストのみ・画像添付なし）
```

### 洗面台（Q1〜Q7）

```
Q1 ボウルタイプ：ベッセル / オンボウル / 埋め込み / ボウル一体型
Q2 希望幅：600mm / 750mm / 900mm / 1200mm / 自由入力
Q3 収納タイプ：引き出し / 開き扉 / オープン（扉なし） / なし
Q4 水栓タイプ：壁付き / 台付き
Q5 ホース引き出し：必要 / 不要
Q6 自動水栓：必要 / 不要
Q7 その他ご要望（テキストのみ・画像添付なし）
```

### トイレ（Q1〜Q5）

```
Q1 メーカー：TOTO / LIXIL / Panasonic / おまかせ
Q2 タンク：タンクあり / タンクレス / こだわりなし
Q3 手洗い（条件分岐あり）
　 タンクあり → タンク上手洗い / 独立手洗い / 手洗いなし（3択）
　 タンクレス → 独立手洗い / 手洗いなし（2択）
　 こだわりなし → スキップ
Q4 自動開閉：あり / なし
Q5 その他ご要望（テキストのみ・画像添付なし）
```

## 設計方針（重要）

- **機能的な希望のみを聞く**（グレード・素材など見た目に関わる項目は聞かない）
- **水回りはコストを抑える**方針（予算はLDKに集中させる）
- **画像添付はキッチンのみ**（水回りで画像を見せるとこだわりが生まれるため）
- 「現状のまま」を選んだ設備はデザイナー用フォームで「⚠️ 要現地確認」と表示する
